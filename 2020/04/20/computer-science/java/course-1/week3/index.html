# JavaSE 基础

# 一丶继承

  面向对象编程中，继承是一个比较重要的概念，继承描述的是事物之间的所属关系，是从已有的类中构建出新的类，已有的类称为父类，新的类称为子类，子类具有父类的数据属性和方法，并能扩展新的属性和方法（不能继承私有）。

<p align="left">1、继承的使用

 格式：使用 extends 关键字 ==> class 子类 extends 父类 ()

 创建： Person 类和 Student 子类：public class Student extends Person

2、继承的优点

 提高了代码的复用性，提高了软件的开发效效率；
 让类与类之间产生了联系，提供了多态的前提；

3、继承的特点

 在 Java 中，只支持单继承，不支持多继承，一个子类只能有一个直接父类；
 父类私有成员，子类不能继承，因为根本看不到；
 一个父类可以有多个子类；
 支持多层继承，一个类的父类可以再去继承另外的父类；
 子类和父类是相对的概念，一个类是某个类父类的同时，也可以是另一个类的子类。

4、继承中成员变量关系

 不同变量名子类直接调用；
 相同变量名调用时，如果子类有，就不调用父类的变量，如果子类没有，父类有，则调用父类的；
 在子类中，要调用父类的成员，使用关键字 super，调用父类方法也是一样，使用 super 关键字。
</p>

<li> 在子类中，要调用父类的成员，使用关键字 super，调用父类方法也是一样，使用 super 关键字 </li>
</ul>

# 二丶抽象

  当父类知道子类应该包含什么样的方法，但无法确定子类如何实现这些方法；在分析事物时，会发现事物的共性，将共性抽取出，实现的时候，就会有这样的情况：方法功能声明相同，但方法功能主体不同，这时，将方法声明抽取出，那么，此方法就是一个抽象方法。

1. 抽象的定义格式
<ul>
<li> 抽象方法的定义格式：public abstract 返回值类型 方法名（参数）；</li>
<li> 抽象类的定义格式：abstract class 类名 {}</li>
</ul>

2. 抽象的特点
<ul>
<li> 抽象类和抽象方法都需要被 abstract 修饰，抽象方法一定要定义在抽象类中 </li>
<li> 抽象不能直接创建对象，因为调用抽象方法没有意义 </li>
<li> 只有覆盖了抽象类中所有的抽象方法后，其子类才可以创建对象，否则该子类还是一个抽象类 </li>
<li> 之所以继承抽象类，更多的是在思想，是面对共性类型操作会更简单 </li>
</ul>

3. 抽象的注意事项
<ul>
<li> 抽象类一定是个父类，因为是不断抽取而来的 </li>
<li> 抽象类中可以不定义抽象方法，其存在的意义就是不让该类创建对象，方法可以直接让子类去使用 </li>
<li> 抽象关键字 abstract 不可以和以下关键字共存：
private：私有的方法子类是无法继承到的，也不存在覆盖，如果 abstract 和 private 一起使用修饰方法， abstract 既要子类去实现这个方法，而 private 修饰子类根本无法得到父类这个方法，互相矛盾
final：final 修饰的类不能被继承，而抽象类一定是父类
static：static 修饰的表示静态的，不能被修改的，但可以直接被类所调用，而 abstract 修饰的是抽象的，即没有方法实体，也不能直接被调用 </li>
</ul>

<li> 重写父类的抽象方法时，去掉 abstract 关键字，加上方法主体 </li>
</ul>

# 三丶接口

 接口表示的是功能的集合，可看做是一种数据类型，接口中全是抽象方法，没有普通方法，是比抽象更抽象的 “类”，接口只描述应该具备的方法，并没有具体实现，具体的实现由接口的实现类（相当于接口的子类）来完成。这样将功能的定义与实现分离，优化了程序设计，解决了继承带来的耦合性，是一种只包含了功能声明的特殊类。

1. 接口的定义
<ul>
<li> 与定义类的 class 不同，使用 interface 关键字定义接口 </li>
<li> 定义格式：
public interface 接口名 {
抽象方法 1；
抽象方法 2；
抽象方法 3；
}</li>
<li> 使用 interface 代替了原来的 class，其他步骤与定义类相同 </li>
</ul>
逻辑实例：

2. 类实现接口
<ul>
<li> 类与接口的关系为实现关系，即类实现接口，实现的动作类似继承，只是关键字不同，实现使用 implements </li>
<li> 其他类（实现类）实现接口后，相当于声明：“我应该具备这个接口中的”。实现类仍然需要重写方法实现具体的功能 </li>
<li> 格式：
class 类 implements 接口 {
重写接口中方法
}</li>
</ul>

逻辑实例：

3. 接口特点
<ul>
<li> 在类实现接口后，该类就会将接口中的抽象方法继承过来，此时该类需要重写该抽象方法，完成具体的逻辑 </li>
<li> 接口中定义功能，当需要有该功能时，可以让类实现该接口，接口只声明了应该具备的方法，是功能的声明 </li>
<li> 在具体实现类中重写方法，实现功能，是方法的具体实现 </li>
<li> 通过类实现接口将功能的声明与实现分离开来 </li>
<li> 接口中的方法均为公共访问的抽象方法 </li>
<li> 接口中无法定义普通的成员变量 </li>
</ul>

4. 接口中成员的特点
<ul>
<li> 接口中可以定义变量，但是变量必须有固定的修饰符修饰，public static final  接口中的变量也称之为常量，其值不能改变，接口中的变量默认有 public static final 修饰符
eg：public static final int NUM = 1；        //NUM 的值不能改变 </li>
<li> 接口中可以定义方法，方法也有固定的修饰符：public abstract
eg：public abstract void fun();</li>
<li> 接口不可以创建对象 </li>
<li> 子类必须覆盖掉接口中所有的抽象方法后，子类才可以实例化，否则子类还是一个抽象类 </li>
</ul>

5. 接口的多实现
 Java 是不支持多继承的，因为当多个父类有相同的功能时，子类调用会产生不确定性，其核心原因就是在多继承父类中功能有主体，而导致调用运行时不确定运行哪个主体内容，采用接口的多实现很好的解决了这个问题，因为接口中没有方法体，由子类来明确。

逻辑实例：

6. 接口的多继承
<ul>
<li> 类和类之间通过继承产生关系，接口和类之间通过实现产生关系，而接口和接口之间也是通过继承产生关系。多个接口之间使用 extends 进行继承。</li>
<li> 在 Java 中是不支持多继承的，但接口可以多继承，其核心原因和接口的多实现一样 </li>
</ul>

7. 类继承类同时实现接口
子类通过继承父类拓展功能，如果子类想要继续拓展其他类中的功能，可以通过接口来完成。

逻辑实例：

8. 接口的思想
<ul>
<li> 接口的出现拓展了功能 </li>
<li> 接口其实就是暴露出来的规则 </li>
<li> 接口的出现降低的耦合性 </li>
<li> 接口的出现方便后期的维护 </li>
</ul>

9. 接口和抽象对比
<font color="#dd0000">
接口和抽象的相同点：
<ul>
<li> 都位于继承的顶端，用于被其他类实现或继承 </li>
<li> 都不能直接实例化对象 </li>
</ul>
接口和抽象的区别：
<ul>
<li> 抽象类为部分方法提供实现，避免子类重复使用这些方法，提高代码重用性；接口只能包含抽象方法 </li>
<li> 一个类只能继承一个直接父类（可能是抽象类），但可以实现多个接口（接口弥补了 Java 的单继承）</li>
<li> 抽象类是这个事物中应该具备的内容 </li>
<li> 接口是这个事物中额外的内容 </li>
<li> 接口的实现类必须全部重写父类的抽象方法 </li>
<li> 抽象类可以不用全部重写父类的方法：</li>
<li> 一个抽象类的子类可以是没有抽象方法的实体类，也可以是有抽象方法的抽象类，对于实体类就是全部覆盖了父类的抽象方法，抽象类则是在父类的基础上继续抽象，可以有更多的抽象方法，让子类实现的选择就更多 </li>
<li> 抽象类可以没有抽象方法，这样就和其他类没有区别，只是不能实例化 </li>

</ul>
二者的选用：
<ul>
<li> 优先选用接口，尽量少用抽象类 </li>
<li> 需要定义子类的行为，又要为子类提供共性功能时才选用抽象类 </li>
</ul>
</font><br />

# 四丶多态

 1、概述：多态就是描述一个事物的多种形态，如小明是一个学生，他既是人，也是学生，出现两种形态。Java 中多态的代码体现在一个子类对象 (实现类对象) 既可以给这个子类 (实现类对象) 引用变量赋值，又可以给这个子类 (实现类对象) 的父类 (接口) 变量赋值。

2、多态的定义与使用格式
父类的引用变量指向子类对象：<font color="#dd0000"> 父类类型 变量名 = new 子类类型（）；
变量名。方法名（）；</font><br />

<ul>
<li> 普通类多台定义格式：父类 变量名 = new 子类（）；</li>
</ul>

<ul>
<li> 抽象类定义多态格式： 抽象名 变量名 = new 抽象子类（）；</li>
</ul>

<ul>
<li> 接口多态定义的格式：接口 变量名 = new 接口实现类（）；</li>
</ul>

3、多态的特点
<ul>
<li> 必须有父子类关系或类实现接口关系，否则无法完成多态。</li>
<li> 必须重写父类或接口类方法。</li>
<li> 父类引用变量指向子类对象。</li>
<li> 父类引用变量调用方法时，会调用子类重写后的方法。</li>
</ul>

4、多态成员的特点
<ul>
<li> 成员变量在编译时，参考的是引用类型所属的类（父类）中是否有被调用的成员变量。没有则编译失败。</li>
<li> 成员变量在运行时，参考的也是引用类型所属的类（父类）中是否有被调用的成员变量。没有则运行失败。</li>
</ul>

  • <font color="#dd0000"> 即：编译和运行都是看等号的左边（父类）</font><br />
    <ul>
    <li> 成员方法编译时，参考引用变量所属的类（父类），如果类中没有调用的方法，编译失败。</li>
    <li> 成员方法运行时，参考引用变量所指的对象所属的类（子类），并运行对象所属类中的方法。</li>
    </ul>
    <font color="#dd0000"> 即：编译看左边（父类），运行看右边（子类）</font><br />

5、instanceof 关键字
<ul>
<li> 作用：instanceof 关键字用来判断某个对象是否属于某种数据类型。</li>
<li> 格式：boolean b = 对象 数据类型；</li>
</ul>

6、多态转型
<ul>
<li> 当子类对象赋值给一个父类引用时，即向上转型（多态本身就是向上转型的过程）</li>
<li> 格式：父类类型 变量名 = new 子类类型（）；
eg：Fu f = new Zi();</li>
</ul>
向下转型
<ul>
<li> 通过强制类型转换格式，将父类引用转为子类格式。</li>
<li> 格式：子类类型 变量名 = （子类类型）父类类型的变量；
eg：Zi z = (Zi) f; // 变量 f 实际上是指向 Zi 对象 </li>
</ul>
7、多态的优缺点
优点：
<ul>
<li> 提高了代码的维护性（由继承保证）</li>
<li> 提高代码复用性（有继承保证）</li>
<li> 提高了代码的扩展性：新增加的子类不影响已存在类的多态性、继承性以及其他特性。</li>
<li> 安全性：向上转型将子类类型隐藏了起来。</li>
</ul>
缺点：
<ul>
<li> 只能使用父类共性的内容，无法使用子类特有的功能（如要使用就要使用向下转型强制类型转换）</li>
